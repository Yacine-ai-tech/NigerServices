# Security Policy

## Supported Versions

We currently support the following versions of Niger Services:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of Niger Services seriously. If you believe you have found a security vulnerability, please report it to us by emailing siddoyacinedigitl@gmail.com.

Please include the following information in your report:

- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit the issue

We will acknowledge receipt of your vulnerability report within 48 hours and provide a timeline for resolution. We ask that you do not disclose the vulnerability publicly until we have had a chance to address it.
